🧪 数据过滤器集成功能测试

验证 DataFilterInput 组件与数据源配置表单的集成效果

📋 实现的功能清单

  • JSON示例自动设置为默认值
  • 数据过滤器配置区域动态显示/隐藏
  • JSONPath语法支持（$.data.list, $.users[0], $.result.items）
  • 实时数据过滤和结果预览
  • 配置器逻辑（UI配置）与执行器逻辑（数据应用）分离
  • 过滤路径规则保存到数据项配置中
  • 多数据类型支持（json/http/websocket）

🔧 JSON默认数据示例

{ "deviceInfo": { "id": "device_001", "name": "智能传感器", "status": "online", "location": { "building": "A栋", "floor": 3, "room": "301" } }, "metrics": [ { "type": "temperature", "value": 23.5, "unit": "°C", "timestamp": "2024-01-15T10:30:00Z" }, { "type": "humidity", "value": 65.2, "unit": "%", "timestamp": "2024-01-15T10:30:00Z" } ], "config": { "enabled": true, "reportInterval": 60, "thresholds": { "temperature": { "min": 18, "max": 28 }, "humidity": { "min": 40, "max": 80 } } } }

🎯 测试步骤

步骤 1: 访问测试页面

在开发服务器中打开: /views/test/data-source-config-form-test

或者通过菜单: 测试 → 数据源配置表单测试

步骤 2: 测试JSON默认值

1. 点击左侧配置表单中的 "添加数据项" 按钮

2. 确认JSON输入框已自动填充复杂的示例数据

3. 验证JSON格式正确且包含设备信息、度量数据、配置等多层结构

步骤 3: 测试数据过滤功能

1. 在弹窗中点击 "添加过滤器" 按钮

2. 验证数据过滤器配置区域出现

3. 测试过滤路径，例如:

  • $.deviceInfo.name → 过滤设备名称
  • $.metrics[0] → 过滤第一个度量数据
  • $.config → 过滤配置对象

步骤 4: 验证过滤预览功能

1. 确认数据过滤器显示实时状态提示

2. 修改JSON内容，验证过滤源数据自动更新

3. 输入/修改过滤路径，验证过滤结果正确

步骤 5: 测试配置保存

1. 设置数据过滤路径后点击 "确认添加"

2. 验证数据项列表中显示新添加的项目

3. 点击 "查看" 按钮，确认过滤路径配置已保存

🔍 验证要点

配置器与执行器分离

配置器部分: 弹窗中的UI配置，包括字段映射规则设置

执行器部分: 实际数据处理和字段映射应用（在 handleMappingResultChange 中）

验证方法: 检查控制台输出，确认配置和执行的日志分离

多数据类型兼容性

测试切换到 HTTP/WebSocket 类型时，字段映射功能仍然可用

验证不同数据类型的配置结构正确保存
