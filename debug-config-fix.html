🔧 配置传递修复调试

❌ 发现的问题

根本问题：我在 extractComponentConfig 中错误地替换了整个配置对象，导致丢失了其他配置属性。

// ❌ 错误做法（导致配置丢失） if (hasCustomizeConfigData) { configData = customizeConfig // 只保留了customize部分！ } // ❌ 错误做法（同样有问题） configData = mergedConfig // 只包含root+customize，丢失其他配置

✅ 修复方案

正确做法：保留所有配置属性，同时将customize部分扁平化到根级别。

// ✅ 正确做法（保留所有配置） if (hasCustomizeConfigData) { configData = { ...props.config, // 保留所有原始配置 ...customizeConfig // customize属性扁平化到根级别 } } // ✅ 正确做法（扁平化现有配置中的customize） if (hasCustomizeConfigData) { configData = { ...configData, // 保留所有现有配置 ...customizeConfig // customize属性扁平化到根级别 } }

🔍 问题原因分析

原始配置可能包含： { title: "组件标题", content: "组件内容", fontSize: 14, customize: { themeColor: "#ff0000" } } ❌ 之前的错误处理： configData = customizeConfig // 结果：{ themeColor: "#ff0000" } ← 丢失了title、content、fontSize！ ✅ 现在的正确处理： configData = { ...props.config, ...customizeConfig } // 结果：{ title: "组件标题", content: "组件内容", fontSize: 14, themeColor: "#ff0000" }

  • 配置完整性：所有配置属性都应该正确传递给组件
  • 扁平化效果：customize中的属性应该可以直接访问
