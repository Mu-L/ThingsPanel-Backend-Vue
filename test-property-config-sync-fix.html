修复属性值变更无法触发配置系统更新的关键问题

🎯 根本问题和修复方案

🚨 根本问题确认

用户修改组件属性时，只更新了 Visual Editor Store 中的 node.properties，但没有同步到配置管理系统，导致 ConfigEventBus 没有接收到配置变更事件，因此不会触发 HTTP 数据源重新执行。

✅ 关键修复完成

在 DataFlowManager.handleUpdateNode 中添加了配置系统同步逻辑，确保属性变更时同时更新配置对象，触发完整的事件链路。

🔧 核心修复内容

✅ DataFlowManager.handleUpdateNode 增强

在处理 UPDATE_NODE 操作时，检测到 properties 更新后，自动调用 syncNodePropertiesToConfiguration 方法

// 🔥 关键修复：如果更新包含properties，同时更新配置系统 if (action.data && action.data.properties) { // 获取更新后的完整节点数据 const updatedNode = this.store.nodes.find(n => n.id === action.targetId) if (updatedNode) { // 将节点的properties同步到配置系统的component配置中 await this.syncNodePropertiesToConfiguration(action.targetId, updatedNode.properties) } }

✅ syncNodePropertiesToConfiguration 方法

新增专门的配置同步方法，将节点属性更新到配置系统的 component 配置节中

// 使用updateConfigurationSection触发配置变更事件 this.configService.updateConfigurationSection(componentId, 'component', { ...properties // 将所有properties作为component配置 })
