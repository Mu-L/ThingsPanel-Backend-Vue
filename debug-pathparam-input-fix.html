🔍 路径参数输入框修复验证

专门测试URL配置中的pathParam输入框焦点丢失问题

📍 问题定位结果

您提到的URL参数输入框位于：HttpConfigStep1.vue → DynamicParameterEditor 组件

参数类型：parameter-type="path"，这是专门配置路径参数(pathParam)的输入框

好消息：我的修复已经覆盖了这个组件，应该已经解决了焦点丢失问题！

⚠️ 重要测试步骤

关键步骤1：选择内部接口

1. 访问数据源系统测试页面
2. 在HTTP配置的第1步中，选择"地址类型"为"内部地址"
3. 选择任何一个有路径参数的内部接口（如：设备详情接口 - /device/detail/{id}）

关键步骤2：启用URL传参

1. 选择内部接口后，会出现"URL传参"开关
2. 打开URL传参开关
3. 此时会显示"参数配置"区域，包含pathParam输入框

关键步骤3：测试参数名输入框

1. 在"参数配置"区域，找到参数名输入框（默认可能是"pathParam"或空）
2. 连续输入多个字符测试焦点保持
3. 验证：输入框应该保持焦点，不会每输入一个字符就失去焦点

✅ 修复的技术细节

// HttpConfigStep1.vue 中的路径参数配置 <DynamicParameterEditor :model-value="urlParams" parameter-type="path" // 这是路径参数 add-button-text="添加URL参数" key-placeholder="参数名（如：id）" // 这就是您提到的参数名输入框 value-placeholder="参数值" :max-parameters="1" @update:model-value="onUrlParamsUpdate" />

修复内容：

  • 稳定的Vue Key值：:key="param._id"
  • 优化的事件处理：@input 替代 @update:value
  • 智能默认值策略：仅在失去焦点且为空时设置默认值
  • 历史参数兼容：自动为旧参数生成稳定ID
