🔧 高度计算修复验证测试

验证 PanelLayout 高度计算逻辑修复是否解决了滚动问题

🔍 问题分析

根据用户提供的 DOM 结构分析，发现高度计算产生了错误的负值：

❌ 错误的高度计算
height: calc(-128px + 100vh)

这导致了嵌套层级的高度继承问题，滚动容器无法正确显示滚动条。

✅ 修复方案

  1. 修复高度计算逻辑：直接检查 props 和 slots 而不依赖 computed 条件判断
  2. 添加安全检查：确保 totalFixedHeight 和 actualTotal 不为负数
  3. 简化条件逻辑：避免复杂的显示条件计算导致的错误
  4. 增加调试日志：输出关键计算值便于问题排查
✅ PanelLayout.vue 修复
- dynamicHeights 计算逻辑修复
- 使用 Math.max(0, totalFixedHeight) 防止负值
- 直接检查 props.showXX && slots.xx
✅ 高度传递链修复
PanelLayout → PanelEditorV2 → GridLayoutPlusWrapper
正确的 availableHeight 属性传递
✅ CSS 样式优化
- .renderer-main-area: overflow: visible
- .renderer-container: height: auto
- .main-area: overflow: auto (唯一滚动容器)
⚠️ 需要验证
1. 空状态不显示滚动条
2. 添加组件后正确显示滚动条
3. 拖拽调整大小后滚动正常
4. DOM 显示正确的 calc(100vh - XXXpx)

🎯 预期结果

  • 控制台输出正确的高度计算值（totalFixedHeight, actualTotal, availableHeight）
  • DOM 中显示 height: calc(100vh - [正数]px) 而不是负值
